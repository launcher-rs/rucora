# 工具调用改进研究报告

## 执行摘要

本报告分析了 agentkit 当前工具调用系统的缺点，并提出系统性改进方案。

## 1. 当前系统架构

### 1.1 核心组件
- **ToolRegistry**: 工具注册表，管理工具的注册、查询、过滤
- **ToolExecution**: 工具执行模块，负责实际执行工具调用
- **DefaultExecution**: 聚合执行能力，管理工具调用循环
- **MiddlewareChain**: 中间件链，支持前后处理钩子

### 1.2 工具调用流程
```
LLM Response → ToolCall → Policy Check → Tool Execution → Result Processing
```

## 2. 发现的缺点

### 2.1 可靠性问题
| 问题 | 影响 | 优先级 |
|------|------|--------|
| 无重试机制 | 临时失败导致任务中断 | P0 |
| 无超时控制 | 耗时工具阻塞执行 | P0 |
| 无熔断机制 | 失败工具持续消耗资源 | P1 |

### 2.2 控制粒度问题
| 问题 | 影响 | 优先级 |
|------|------|--------|
| 并发控制粗粒度 | 无法针对特定工具限制 | P1 |
| 无工具链支持 | 无法串联依赖工具 | P2 |
| 无条件调用支持 | 无法根据结果决定下一步 | P2 |

### 2.3 可观测性问题
| 问题 | 影响 | 优先级 |
|------|------|--------|
| 观测器事件不完整 | 调试困难 | P2 |
| 缺乏执行进度 | 用户体验差 | P2 |

### 2.4 扩展性问题
| 问题 | 影响 | 优先级 |
|------|------|--------|
| 无版本管理 | 无法动态更新工具 | P2 |
| 无结果缓存 | 重复调用浪费资源 | P2 |

## 3. 改进方案

### 3.1 P0 改进：可靠性

#### 3.1.1 工具重试机制
- 为工具配置重试次数 (默认 3 次)
- 指数退避策略 (可选)
- 可配置的重试条件 (仅 transient errors)

#### 3.1.2 超时控制
- 为工具配置超时时间 (默认 30s)
- 可针对不同工具设置不同超时
- 超时后触发重试/降级

#### 3.1.3 熔断机制
- 工具失败计数统计
- 连续失败达到阈值自动熔断
- 熔断后定期探测恢复

### 3.2 P1 改进：控制粒度

#### 3.2.1 细粒度并发控制
- 支持按工具名称设置独立并发数
- 支持按工具分类设置并发限制

#### 3.2.2 工具链支持
- 支持声明工具依赖关系
- 自动按依赖顺序执行

### 3.3 P2 改进：可观测性

#### 3.3.1 增强观测器事件
- 工具执行开始/结束事件
- 重试事件
- 熔断状态变化事件
- 进度报告事件

### 3.4 P2 改进：扩展性

#### 3.4.1 工具版本管理
- 支持工具动态更新描述
- 支持工具热更新

#### 3.4.2 工具结果缓存
- 基于输入 hash 缓存结果
- 可配置 TTL

## 4. 实现计划

### Phase 1: 可靠性改进
1. 添加 RetryConfig 和 TimeoutConfig
2. 实现重试逻辑
3. 实现超时控制
4. 实现熔断器

### Phase 2: 控制改进
1. 增强并发控制配置
2. 添加工具依赖支持

### Phase 3: 可观测性
1. 增强观测器事件
2. 添加进度报告

### Phase 4: 扩展性
1. 添加版本管理
2. 实现结果缓存

## 5. 兼容性考虑

- 所有新配置都有默认值，保持向后兼容
- 新功能默认关闭，通过配置启用
- 不改变现有 API 签名
